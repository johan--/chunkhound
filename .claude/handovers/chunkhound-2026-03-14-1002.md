## Key Findings (Code Analysis)
- **Root cause confirmed:** `_cleanup_orphaned_files()` (indexing_coordinator.py:2775) calls `delete_file_completely()` in a Python loop, one file at a time
- `_executor_delete_file_completely()` (duckdb_provider.py:1251) does raw DELETEs against live HNSW-indexed embedding tables — no index drop/recreate, no transaction wrapper
- The safe bulk write path (`embedding_repository.py:94`) does: drop HNSW → bulk mutate → recreate HNSW
- The deletion path has none of this index awareness — that's the gap
- Search handlers catch all exceptions and return empty results — silent failure mode
