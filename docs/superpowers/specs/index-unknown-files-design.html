Design: index_unknown_files Flag

Date 2026-05-13
Status Approved — pending implementation
Approach C — flag + binary guard

🐛 Problem

In v5, files with unrecognized extensions (Dockerfile, .proto, .feature, etc.) are silently skipped at batch_processor.py:224 when language == Language.UNKNOWN. Before v5 a bug accidentally indexed these as plain text — users found this useful. The fix removed the behavior; this design adds it back as an explicit, opt-in flag.

🎯 Goal

  • 🔍Unknown-extension files are discovered during the directory scan when the flag is on
  • 📄Discovered unknown files are parsed as plain text via TextMapping
  • 🛡️Binary files (null-byte heuristic) are still skipped to protect the index

🖥️ Customer-Facing Interfaces

All three surfaces expose the flag. MCP tools don't expose indexing params (set at server startup).

Interface
Key / Flag
Default
CLI (index, mcp, _daemon)
--index-unknown-files
off
.chunkhound.json

⚙️ Architecture

1

IndexingConfig — new public field

chunkhound/core/config/indexing_config.py

Add alongside detect_embedded_sql, per_file_timeout_seconds, etc.:

index_unknown_files: bool = Field(
    default=False,
    description="Index files with unrecognized extensions as plain text",
)

Flows automatically into config_dict via the existing model_dump() path — no extra wiring needed.

2

File Discovery — append **/* when flag is on

chunkhound/core/config/indexing_config.py — model_validator(mode="after")

When index_unknown_files=True, append **/* to the include list (if not already present). Applies to both default and custom include lists. Existing exclude patterns (node_modules, .git, .env) still apply.

3

Parsing — binary guard + TEXT fallback

chunkhound/services/batch_processor.py — line 224
# Current (unconditional skip):
if language == Language.UNKNOWN:
    # ... skip with status="skipped"

# New:
if language == Language.UNKNOWN:
    if not config_dict.get("index_unknown_files"):
        # ... skip (unchanged when flag=off)
    else:
        # Binary guard: read first 8 KB, check for null bytes
        try:
            with open(file_path, "rb") as fh:
                sample = fh.read(8192)
            if b"\x00" in sample:
                # ... skip, error="binary_file"
                continue
        except OSError:
            # ... skip, status="error"
            continue
        language = Language.TEXT  # fall through to normal parse path
4

CLI Flag

chunkhound/core/config/indexing_config.py — add_cli_arguments()
parser.add_argument(
    "--index-unknown-files",
    action="store_true",
    default=False,
    dest="index_unknown_files",
    help="Index files with unrecognized extensions as plain text"
         " (binary files are still skipped)",
)

Auto-registered on all three commands via add_config_arguments(..., ["indexing", ...]): index, mcp, _daemon.

5

Environment Variable

chunkhound/core/config/indexing_config.py — load_from_env()
if val := os.getenv("CHUNKHOUND_INDEXING__INDEX_UNKNOWN_FILES"):
    overrides["index_unknown_files"] = val.lower() in ("1", "true", "yes")

🔀 Data Flow

Directory scan
└─ include patterns  ← **/* appended when flag=on
└─ exclude patterns (unchanged)
└─ discovered files
└─ batch_processor.py
├─ Language.known                        → normal parse path (no change)
├─ Language.UNKNOWN + flag=off           → skip (status="skipped")
├─ Language.UNKNOWN + flag=on + binary   → skip (error="binary_file")
└─ Language.UNKNOWN + flag=on + text    → Language.TEXT → TextMapping parse

⚠️ Error Handling

  • OSErrorReading the 8 KB sample fails → skip with status="error"
  • binaryAny null byte in first 8 KB → skip with error="binary_file"; conservative by design
  • log:textlogger.debug("Indexing unknown file as text: {}", file_path)
  • log:binlogger.debug("Skipping binary file: {}", file_path)

🚫 Out of Scope

  • Adding new extensions to the EXTENSION_TO_LANGUAGE map (separate concern)
  • Detecting file encoding (UTF-8 vs Latin-1) — TextMapping handles this gracefully
  • MCP tool parameters — indexing config is set at server startup, not per-tool-call
